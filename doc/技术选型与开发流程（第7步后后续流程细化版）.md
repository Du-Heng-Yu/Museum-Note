## 三、第9步：全流程测试与Bug修复（核心步骤，1-2小时，分阶段执行）

目标：彻底排查所有功能Bug，确保全流程可正常运行，无数据异常、交互异常

推荐模型：Auto（日常小改、Bug修复最省额度；复杂Bug可切换为Claude Sonnet 4.6）

具体操作（分2个阶段，逐一落地）：

### 阶段1：双端模拟器测试（必做，覆盖所有页面和功能）

- 测试环境：iOS模拟器（任意机型）+ Android模拟器（任意机型，建议选择主流机型如Pixel 6）

- 测试流程（按用户使用场景走查）：

    1. 初始化测试：启动App，确认导航正常（Tab切换、Stack页面跳转无报错）、数据库初始化成功

    2. 文物录入测试：通过3种入口（FAB→相册/相机/新建展览）录入文物，确认照片保存正常、年份-朝代联动正常、数据存入数据库

    3. 时间轴测试：查看朝代卡片渲染（颜色、名称、年份）、展开/收起动画、文物卡片预览及跳转

    4. 文物详情测试：查看照片横向滚动、信息展示、编辑功能（修改后数据更新）、删除功能（二次确认、照片清理、数据删除）

    5. 展览管理测试：新建/编辑/删除展览、展览详情查看、文物关联（添加/删除文物）、级联删除（删除展览时，关联文物及照片一并删除）

    6. 异常场景测试：空状态引导（无展览、无文物）、重复操作（重复保存、重复删除）、取消操作（编辑时取消、删除时取消）

- 记录问题：每遇到一个Bug（卡顿、报错、数据异常、布局错位），截图留存，标注具体操作步骤（如“iOS模拟器，文物编辑页，选择相机拍摄后，照片无法显示”）

### 阶段2：Bug修复（逐一处理，修复后立即复测）

- 优先级排序：先修复影响核心功能的Bug（如数据无法保存、删除报错、照片无法存储），再修复次要Bug（如UI细微错位、Toast提示不清晰）

- 修复操作：将Bug截图+具体操作步骤提供给AI，要求生成修复代码，替换项目中对应文件，保存后重新运行测试

- 复测要求：每个Bug修复后，必须回到对应场景复测，确认Bug已解决，且未引入新的问题

验收标准：双端模拟器中，所有功能流程可正常走通，无Bug；数据增删改查正常，照片存储/清理正常；交互流畅，无卡顿、无布局错位。

## 四、第10步：构建与上架准备（最终步骤，分平台执行，1-2天）

目标：完成EAS Build打包，准备应用商店上架所需素材和配置，确保符合平台审核规范

推荐模型：Claude Opus 4.6（上架流程涉及配置细节和审核规范，长链路推理更稳妥）

具体操作（分3个环节，逐步推进）：

### 环节1：EAS Build打包配置（1-2小时）

- 安装EAS CLI（若未安装）：执行 `npm install -g eas-cli`

- 配置 `eas.json`：向AI提供项目现有配置，要求生成符合双端打包的 `eas.json`（包含构建环境、签名配置等）

- 执行打包命令：分别打包iOS和Android版本，命令为 `eas build --platform ios` 和 `eas build --platform android`

- 等待打包完成：打包成功后，会获得对应平台的安装包（iOS为.ipa，Android为.apk/aab）

### 环节2：应用商店上架素材准备（重点，半天）

- 通用素材：App图标（不同尺寸，符合双端要求）、App名称、简介文案、功能截图（iOS需提供不同机型截图，Android需提供主流机型截图）

- 隐私政策：生成纯本地存储的隐私政策（明确说明“App仅在本地存储用户数据，不采集、不上传任何用户信息”），可让AI生成后修改完善

- 权限声明：准备相机、相册权限的说明文案（如“相机用于拍摄文物照片，相册用于选择文物照片，所有照片仅存储在本地”），用于上架时填写

### 环节3：应用商店提审（半天）

- iOS平台（App Store Connect）：
        

    1. 注册/登录App Store Connect账号，创建App（填写App名称、简介、隐私政策链接等）

    2. 上传打包好的.ipa文件、功能截图、权限声明

    3. 填写提审信息，提交审核（审核周期约1-3天）

- Android平台（Google Play Console）：
        

    1. 注册/登录Google Play Console账号，创建应用，填写应用信息

    2. 上传打包好的.aab文件、功能截图、隐私政策、权限声明

    3. 完成内容分级、定价等设置，提交审核（审核周期约1-7天）

验收标准：EAS Build打包成功，双端安装包可正常安装使用；应用商店提审材料齐全，提交审核无报错。
